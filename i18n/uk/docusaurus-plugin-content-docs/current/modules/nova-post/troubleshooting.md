# Усунення несправностей

## Типові проблеми та рішення

### Проблема: «Нова Пошта» не відображається як спосіб доставки на сайті

**Симптоми:** Під час оформлення замовлення на вебсайті «Нова Пошта» відсутня серед доступних способів доставки.

**Можливі причини:**
- Спосіб доставки «Нова Пошта» не опублікований.
- Не заповнені Місто відправника, Відділення або Ключ АПІ.
- Вага/об'єм кошика перевищує обмеження, встановлені на вкладці **Вміст**.
- Теги товарів не відповідають правилам **Обов'язкові / Виключені теги** способу доставки.

**Рішення:**
1. Перейдіть у **Продажі** або **Веб-сайт** ➔ **Конфігурація** ➔ **Способи доставки**.
2. Відкрийте спосіб **Нова Пошта** та перевірте вкладку **Налаштування** — мають бути заповнені **Ключ АПІ**, **Місто відправника**, **Відділення** та **Номер телефону відправника** (формат `380XXXXXXXXX`).
3. На вкладці **Вміст** переконайтеся, що **Макс. вага / Макс. об'єм** дорівнюють `0.00` або перевищують значення кошика, а правила тегів дозволяють поточні товари.
4. Опублікуйте спосіб доставки та оновіть кошик на сайті.

### Проблема: «Рахунок для виплати накладного платежу обов'язковий, коли увімкнено накладний платіж.»

**Симптоми:** Під час збереження способу доставки з увімкненим накладним платежем з'являється помилка валідації.

**Можливі причини:**
- Увімкнено **Накладний платіж**, але не заповнено **Рахунок для виплати накладного платежу**.

**Рішення:**
1. У **Продажі** ➔ **Конфігурація** ➔ **Способи доставки** відкрийте спосіб «Нова Пошта».
2. У блоці **Накладний платіж** заповніть поле **Рахунок для виплати накладного платежу** — банківський рахунок компанії у форматі IBAN.
3. Якщо рахунок ще не створено, спочатку додайте його у **Контакти** ➔ ваша компанія ➔ вкладка **Бухоблік**, а потім поверніться до способу доставки.

:::info Формат номера телефону
Номер телефону відправника валідується у форматі `380XXXXXXXXX` — рівно 12 цифр, що починаються з `380`. Будь-який інший формат викликає помилку: *Номер телефону має бути у форматі 380XXXXXXXXX (12 цифр). Приклад: 380441234567*.
:::

### Проблема: «Доставка Новою Поштою вимагає налаштування типу відправника та типу отримувача.»

**Симптоми:** Натискання **Отримати ставку** в помічнику доставки (**Додати доставку**) призводить до помилки валідації.

**Можливі причини:**
- На способі доставки не обрано **Тип відправника**.
- На контакті адреси доставки не обрано **Тип отримувача** (Відділення/Поштомат).

**Рішення:**
1. Відкрийте спосіб доставки «Нова Пошта» та переконайтеся, що на вкладці **Налаштування** обрано **Тип відправника**.
2. Відкрийте контакт адреси доставки в замовленні та встановіть поле **Тип отримувача** (Відділення/Поштомат).
3. Поверніться до помічника та натисніть **Отримати ставку** знову.

### Проблема: Експрес-накладну (ТТН) не створено при натисканні «Підтвердити»

**Симптоми:** Натискання **Підтвердити** на складському переміщенні не генерує ТТН — з'являється помилка валідації.

**Можливі причини:**
- *Доставка Новою Поштою вимагає пов'язаного замовлення з датою відправки.* — у замовленні на продаж не вказано **Дату відправлення**.
- *Перевізник Нової Пошти … не має налаштованого типу відправника.* — у перевізника не обрано тип відправника.
- *Адреса доставки … не має налаштованого типу отримувача.* — у партнера не обрано тип отримувача.
- *Тип вантажу в основному записі та його рядках несумісний.* — рядки опцій місць не відповідають обраному **Типу вантажу** (Документи/Вантаж/Посилка/Паллет).
- *Потрібно заповнити хоча б одне значення для місць* — у помічнику не додано жодного рядка місця.
- Помилка API, повернута «Новою Поштою» (невірний референс адреси, проблема з API-ключем тощо).

**Рішення:**
1. Відкрийте замовлення на продаж та заповніть **Дату відправлення**.
2. Перевірте **Тип відправника** на способі доставки та **Тип отримувача** на контакті адреси доставки.
3. У помічнику доставки додайте хоча б один рядок місця/пакунка, тип якого відповідає **Типу вантажу**.
4. Якщо помилка посилається на повідомлення API — відкрийте **Склад** ➔ **Нова Пошта** ➔ **Коди помилок API**, щоб побачити розшифрований опис від «Нової Пошти», та виправте відповідні дані.

### Проблема: Ручне додавання ТТН відхилено — «Номер ТТН повинен бути унікальним» або «Кожне переміщення може мати лише одну ТТН»

**Симптоми:** Введення 14-значного номера у поле **Номер відстеження** завершується однією з цих помилок.

**Можливі причини:**
- Цей самий номер ТТН вже зареєстровано в Odoo на іншому переміщенні.
- До поточного переміщення вже прив'язано накладну — додати другу не можна.

**Рішення:**
1. Відкрийте **Нова Пошта** ➔ **Накладні** та знайдіть номер ТТН.
2. Якщо накладна вже існує — зв'яжіться з наявним записом або спершу видаліть/замініть ТТН на початковому переміщенні.
3. Щоб замінити ТТН на поточному переміщенні, очистіть поле **Номер відстеження** та збережіть — система автоматично видалить попередню накладну — а потім введіть новий номер.

### Проблема: У селекторі відсутні міста або відділення

**Симптоми:** Неможливо знайти потрібне місто або відділення під час вибору адреси отримувача.

**Можливі причини:**
- Щоденна дія CRON для синхронізації довідників «Нової Пошти» ще не виконувалася або завершилася з помилкою.
- Місто/відділення було додане на стороні «Нової Пошти» після останньої синхронізації.

**Рішення:**
1. Перейдіть у **Склад** ➔ **Операції** ➔ **Нова Пошта** та перевірте списки **Міста** / **Відділення**.
2. Запустіть заплановану дію вручну: **Налаштування** ➔ **Технічні** ➔ **Заплановані дії**, знайдіть дію синхронізації «Нової Пошти» та натисніть **Запустити вручну**.
3. Якщо синхронізація продовжує завершуватися помилкою, перевірте, що **Ключ АПІ** на способі доставки є дійсним і не прострочений.
