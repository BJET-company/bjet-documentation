# Використання модуля

## Оплата з вебсайту Odoo

### Послідовність кроків для оплати:

1. Перейдіть на сайт Odoo.
2. Додайте товар у кошик.
3. Перейдіть до оформлення замовлення (Checkout).
4. На кроці вибору способу оплати оберіть **LiqPay**.

![Оформлення замовлення на вебсайті — вибір LiqPay](/img/liqpay-payment/ua-website-checkout-liqpay.png)

5. Підтвердіть оформлення — система перенаправить користувача на сторінку LiqPay

![Сторінка оплати LiqPay з варіантами оплати](/img/liqpay-payment/ua-liqpay-payment-gateway.png)

6. Клієнт проводить оплату карткою на стороні LiqPay.

### Результат оплати

#### Результат оплати, якщо модуль Бухоблік встановлено

Якщо модуль **Бухоблік (accountant)** встановлено:

Після успішної оплати:

1. В Odoo створюється **Замовлення на продаж (SO)** у статусі Замовлення на продаж. 
2. Автоматично створюється **Рахунок**

![Замовлення на продаж після оплати з посиланням на рахунок](/img/liqpay-payment/ua-sales-order-after-payment.png)

3. Статус оплати рахунку **В оплаті**

![Рахунок у статусі «В оплаті»](/img/liqpay-payment/ua-invoice-in-payment.png)

4. Створюється пов’язаний Платіж на всю суму оплати зі статусом **В процесі**

![Запис платежу через LiqPay зі статусом «В процесі»](/img/liqpay-payment/ua-payment-record.png)

У полі **Призначення платежу** система зберігає **унікальний ID платежу в системі LiqPay**. Інформація з поля **Призначення платежу** передається на створенні бухгалтерські записи, і може бути використана для узгодження банківських виписок з LiqPay-платежами.

5. Також автоматично створюється **Платіжна операція**, перехід до якої доступний з рахунку через **Розумні кнопки**, так само як і перехід до пов’язаного **Платежу**

![Огляд рахунку та пов'язаного платежу](/img/liqpay-payment/ua-invoice-payment-overview.png)

Для завершення операції необхідно **рознести банківську виписку** та виконати узгодження з цим платежем. Після узгодження статус Рахунку та Платежу зміниться на **Оплачено**

Для спрощення пошуку та звірки платежів:

- у **Payment Transaction** в полі Payment Reference зберігається **унікальний ID платежу в системі LiqPay** (payment_id);
- у пов’язаному **Payment** цей самий ID LiqPay записується в поле **Memo** (разом із, за потреби, стандартним текстом).

Поле **Memo** передається до **Journal Items**, і використовується як основне поле для узгодження банківських виписок з LiqPay-платежами.

#### Результат оплати, якщо модуль Бухоблік не встановлено

Якщо модуль **Бухоблік (accountant)** не встановлено:

Після успішної оплати:

1. В Odoo створюється **Замовлення на продаж (SO)** у статусі Замовлення на продаж.
2. Автоматично створюється **Рахунок** у статусі оплати **Оплачено**
3. Створюється пов’язаний **Платіж** на всю суму оплати зі статусом **Оплачено**.
4. Створюється **Платіжна операція**, перехід до якої доступний з рахунку через розумні кнопки, так само як і перехід до пов’язаного **Платежу**, з інформацією про деталі оплати LiqPay 

Операція продажу вважається завершеною; окремого процесу рознесення банківської виписки немає.

## Оплата через Посилання платежу

### Створення посилання на оплату із Замовлення на продаж або Рахунка

Посилання платежу можна сформувати як з **Замовлення на продаж**, так і з **Рахунка**.

1. Відкрийте потрібне **Замовлення на продаж** або **Рахунок**.
2. Натисніть іконку **налаштувань** та оберіть дію **Створити посилання платежу**

![Створення посилання платежу із замовлення](/img/liqpay-payment/ua-quotation-create-payment-link.png)

3. Система згенерує унікальне посилання на оплату
4. Скопіюйте посилання натиснувши на кнопку Створити та копіювати посилання платежу та надішліть посилання клієнту

![Діалог створення та копіювання посилання платежу](/img/liqpay-payment/ua-payment-link-dialog.png)

### Оплата через Посилання платежу

1. Клієнт переходить за отриманим посиланням.
2. На сторінці оплати обирає метод **LiqPay**.
3. Натискає **Оплатити** — система перенаправляє його на сторінку LiqPay.
4. Клієнт проводить оплату карткою на стороні LiqPay.

Наступні кроки в Odoo (створення Рахунка, Платежа, статуси документів) **аналогічні до оплати з вебсайту**, описаної в розділі 4, і залежать від того, чи встановлено модуль accountant.
