# Усунення несправностей

## Типові проблеми та рішення

### Проблема: Помилка автентифікації з API ПриватБанку

**Симптоми:** При імпорті виписок з'являється помилка: "PrivatBank API authentication failed" або "Access forbidden"

**Можливі причини:**
- Невірний Токен або ID Автоклієнт ПриватБанк24
- Токен не активований або закінчився термін дії
- Невірний формат облікових даних

**Рішення:**
1. Скористайтеся офіційною інструкцією для створення профілю Автоклієнт: [https://docs.google.com/document/d/e/2PACX-1vTtKvGa3P4E-lDqLg3bHRF6Wi9S7GIjSMFEFxII5qQZBGxuTXs25hQNiUU1hMZQhOyx6BNvIZ1bVKSr/pub](https://docs.google.com/document/d/e/2PACX-1vTtKvGa3P4E-lDqLg3bHRF6Wi9S7GIjSMFEFxII5qQZBGxuTXs25hQNiUU1hMZQhOyx6BNvIZ1bVKSr/pub)
2. Скопіюйте і **Токен**, і **ID клієнта** з вашого бізнес-рахунку ПриватБанку
3. Вставте їх у відповідні поля на вкладці ПриватБанк
4. Збережіть журнал і спробуйте імпорт знову

:::warning Важливо
Обидва параметри – Токен І ID Клієнта – є обов'язковими. Модуль не працюватиме, якщо будь-який з них відсутній або невірний.
:::

### Проблема: Вкладка ПриватБанк не з'являється

**Симптоми:** Після вибору "Autoclient PrivatBank24" у полі Bank Feeds вкладка не з'являється.

**Можливі причини:**
- Модуль не встановлений
- Тип журналу не "Банк"

**Рішення:**
1. Переконайтеся, що модуль імпорту виписок ПриватБанку встановлений
2. Перевірте, що тип журналу = "Банк" (Bank)
3. Оновіть сторінку та повторіть вибір

### Проблема: Дублювання транзакцій

**Симптоми:** Одні й ті самі транзакції імпортуються повторно.

**Рішення:**
Модуль автоматично запобігає дублюванню транзакцій. Якщо проблема зберігається:
1. Перевірте, що номер рахунку в журналі збігається з номером у ПриватБанку
2. Зверніться до технічної підтримки
